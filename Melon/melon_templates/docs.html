  1. Introduction

    MelonPY is a lightweight web framework designed for small-scale projects. It offers a simple yet powerful interface for building web applications,complete with routing, templating, session management, and middleware support.

  2. Purpose

    The MelonPY framework is designed to provide a lightweight and flexible platform for building web applications. Its primary goal is to simplify the development process by offering essential features that facilitate rapid application development while maintaining a clean and organized codebase. MelonPY is ideal for small-sized projects, enabling developers to focus on creating functional applications without being overwhelmed by complex configurations.

  3. Features

    • Lightweight: MelonPY is designed to be minimalistic, allowing developers to quickly set up and run their applications without unnecessary overhead.
    • Modular Structure: The framework follows a modular approach, with clearly defined components such as routing, middleware, and error handling, promoting code organization and reusability.
    • Template Rendering: Supports a simple template rendering system with jinja2 to create dynamic HTML pages, making it easier to separate business logic from presentation.
    • Static/dev File Handling: Provides a straightforward way to serve static/dev files (CSS, JavaScript, images) alongside dynamic content.
    • Error Handling: Built-in error handling methods to manage exceptions gracefully, ensuring a better user experience.
    • Middleware Support: Allows developers to implement middleware functions that can process requests and responses, enabling custom logic at different stages of the request lifecycle.
    • Routing: Simple yet powerful URL routing capabilities, allowing developers to map URLs to specific view functions effortlessly.
    • Session Management: Implements basic session functionality to store user-related data across requests, enabling a more personalized user experience.
    • GET Method: Used for retrieving data, supports path parameters and query parameters, and can handle requests with or without parameters.
    • POST Method: Used for sending data to the server, can handle form submissions and JSON data, facilitating resource creation and updates.
    • Extensible: The framework is designed to be easily extensible, allowing developers to integrate additional features or customize existing ones based on project requirements.

    • Session and cookies

      MelonAPP/app.py

                                          
                                    from Melon.app import Mymelon
                                    melon_app = Mymelon()
                                    def example_4(request):
                                        session = getattr(request,"sessions",{ })
                                        visit_count = session.get("visit",0)
                                        visit_count +=1
                                        session["visit_count"] = visit_count
                                        return melon_app.html_response(f'Welcome! You have visited {visit_count} times.')
